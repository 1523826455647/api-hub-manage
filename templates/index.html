添加上游渠道

支持 NewAPI / Sub2API。建议优先使用 Token / Cookie，避免人机验证拦截。

从站点「设置 → API Key」生成的 sk- 密钥选「User API Key」； 管理员面板的系统令牌选「系统令牌」。 User API Key 可查模型但无法查余额/分组，需要完整监控请用 Cookie 登录。
部分站点要求同时提供 User ID。登录站点后 F12 → Application → Cookies → 找 id 或 user_id 字段的值。
1 元 = 多少站内单位？例：1:1 时倍率 0.03 = 1:10 时倍率 0.3，实际成本相同。 该比例用于归一化分组倍率，方便跨渠道对比。
这是实现全自动联动的关键：Sub2API 需要这个 Key 来调用上游的 /v1/chat/completions 接口。 在目标站点登录后 → 个人设置 → API Key / 系统令牌 → 生成一个 sk- 开头的密钥粘贴到这里。 如果你用 Cookie 登录做监控，这个 Key 必须单独填写，Cookie 不能用于上游调用。
